# futu-statement-skill

*[English](README.md) | 中文*

一个 Claude/agent **skill** + 独立 Python 工具，把**富途證券（Futu）香港月结单**
（`證券月結單` PDF）解析成可直接用于报税的 CSV —— 面向**个人所得税·境外所得（个税 境外所得）**、
CRS 申报、以及券商盈亏对账。

**数据只来自 PDF 月结单**，不使用富途 OpenD / API（接口给不出已实现盈亏、手续费，分红也不带标签）。
官方月结单 PDF 才是唯一可信来源。

## 为什么需要它

富途港股月结单是报税的权威凭证，但是：

- **没有「已实现盈亏」栏** —— 必须从成交明细自行重建；
- **没有「派息/股息」栏** —— 现金分红藏在 **`公司行動`（公司行动）** 里；
- 利息、出入金、申赎全都混在 **`資金進出`** 一节中。

本仓库把解析方法沉淀在 [`SKILL.md`](SKILL.md)，并提供一个能自动完成的脚本。

## 安装

```bash
brew install poppler          # macOS（或 apt install poppler-utils）→ 提供 pdftotext
```

无需任何第三方 Python 包（仅标准库）。Python 3.10+ 测试通过。

## 用法

```bash
# 传入单个结单，或者一整个目录
python3 parse_futu_statement.py /结单目录/ -o 输出目录/ --rate 0.90322
```

`--rate` 可选 —— HKD→RMB 的年末中间价；给了就额外输出一列人民币金额。年份会自动从结单中识别。

### 输出文件（UTF-8-BOM，Excel 直接打开不乱码）

| 文件 | 内容 |
|---|---|
| `futu_<年>_成交明细.csv` | 逐笔股票/期权成交；`变动金额` 已扣手续费 |
| `futu_<年>_股息利息现金流.csv` | 分红（`公司行动`）、利息、出入金、各类费用 |
| `futu_<年>_期权行权到期.csv` | 期权到期（EXP）/ 被行权（ASS）事件 |
| `futu_<年>_已实现盈亏_按标的.csv` | 按标的的已实现盈亏（移动加权平均成本法） |
| `futu_<年>_账户净值.csv` | 每月期初/期末资产净值（用于交叉校验） |

脚本运行后会打印 `Σ变动金额`、分红合计、已实现盈亏合计，方便一眼核对。

## 已实现盈亏怎么算

按标的移动加权平均成本，利用富途结单里明确的 `開倉/平倉` 标记。到期/被行权的期权按 0 平掉
（保费全部计为已实现）；年末仍持有的标的已实现记 0（其盈亏属未实现，当年不计税）。可用
`资产净值变动 − 净出入金` 交叉校验。详见 [`SKILL.md`](SKILL.md)。

## 踩坑要点（都是真摔出来的）

- **分红在 `公司行動` 里**，不在「派息」栏；`F/D`=末期息、`I/D`=中期息。
- **`變動金額` 已经是扣费后净额**；每笔手续费 = `|净额| − 成交金额`。
- **方向区分开平/多空**：`買入開倉`/`賣出平倉`/`賣出開倉`/`買入平倉`。
- **行首 `*` 是强制平仓标记**（`*買入平倉 …`），匹配方向前要先剥掉，否则该笔会错挂到上一笔的代码上。
- **利息是成本不是收入**；`基金申購/贖回` 是货基内部腾挪，不算出入金。
- 用 `pdftotext -layout` 提取；期权名称由代码反解，股票名称从持仓段抓取。

## 作为 Claude Code skill 使用

把本目录放进 skills 目录（如 `~/.claude/skills/parsing-futu-statements/`），或用插件管理器安装。
当你让 Claude 解析富途结单、或算富途的盈亏/分红用于报税时，它会自动加载 [`SKILL.md`](SKILL.md)。

## 隐私

仓库内**不含任何个人数据**。脚本只读你指定的 PDF、把 CSV 写到本地；`.gitignore` 已拦截
`*.pdf` 和 `*.csv`，避免误提交结单或结果。

## 可靠性

已用整年 12 份月结单验证：每一笔成交金额、所有标的代码、每个标的的已实现盈亏都完全对得上。
成交数字与股票代码精确无误；期权名称由合约代码反解，股票名称从持仓段获取。
